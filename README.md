# 🌙 Бессонница 2026 — офлайн-программа

Мобильное веб-приложение (PWA) с программой фестиваля анимации **Бессонница 2026**.
Работает **полностью офлайн**, ставится на главный экран телефона как обычное приложение,
показывает описания событий, что идёт прямо сейчас, избранное и **напоминания за 15 минут**
до начала выбранных событий.

## Что умеет

- 📅 **Программа по дням** (9–13 июля) с фильтрами: дневная программа / ночная анимация.
- 🔴 **«Сейчас»** — что идёт в эту минуту и что будет дальше, с обратным отсчётом до старта фестиваля.
- 📖 **Полные описания** событий и списки фильмов в программах показов.
- ⭐ **Избранное** — отметьте событие звёздочкой; список избранного отсортирован по дате и времени.
- ⏰ **Напоминания** приходят за 15 минут (настраивается: 5/10/15/30/60) до начала избранных событий.
- 🔎 **Поиск** по названию, площадке и фильмам.
- 🔄 **Обновление программы** без переустановки: из новой Excel-таблицы (файл или ссылка)
  или проверкой обновления на сервере.
- 📴 **Офлайн-first**: после первого открытия всё живёт в кэше и работает без интернета.
- 📲 **Установка на главный экран** (Android/iOS), запуск в полноэкранном режиме.

## Как пользоваться (для зрителя)

1. Откройте страницу приложения в браузере телефона (см. «Публикация» ниже).
2. Нажмите **⚙️ → «Установить приложение»** (Android) или **«Поделиться» → «На экран Домой»** (iPhone).
3. Один раз откройте при наличии интернета — программа скачается в память телефона.
4. Дальше приложение работает офлайн. Отмечайте события ⭐, разрешите уведомления в настройках —
   и получите напоминание за 15 минут до начала.

## Структура проекта

```
index.html            — разметка приложения
styles.css            — стили (тёмная «ночная» тема)
app.js                — вся логика: рендер, избранное, уведомления, импорт Excel
sw.js                 — service worker (офлайн-кэш + клики по уведомлениям)
manifest.webmanifest  — манифест PWA (установка на главный экран)
data/program.json     — программа фестиваля (генерируется из Excel)
vendor/xlsx.full.min.js — SheetJS, чтение Excel прямо в браузере (для офлайн-импорта)
icons/                — иконки приложения
source_xlsx/          — исходные Excel-файлы фестиваля
scripts/convert_xlsx.py — конвертер Excel → program.json
```

## Обновление программы, когда фестиваль поменял расписание

Есть три пути — все сохраняют ваши отметки «избранное» (id событий стабильны).

**Способ 1. Прямо в приложении (проще всего для зрителя).**
⚙️ → «Загрузить из Excel-файла…» и выберите скачанный с сайта `.xlsx`
(можно выбрать сразу оба файла — программу и анимацию). Или вставьте прямую ссылку на `.xlsx`.
Парсинг Excel происходит в браузере и работает офлайн.

**Способ 2. Пересобрать встроенные данные (для владельца репозитория).**
Положите свежие файлы в `source_xlsx/` (`animation.xlsx` — ночные показы, `nonanimation.xlsx` —
дневная программа) и выполните:

```bash
pip install -r requirements.txt
python3 scripts/convert_xlsx.py
git commit -am "Обновление программы" && git push
```

После публикации зрители получат обновление кнопкой ⚙️ → «Проверить обновление на сервере»
(или автоматически при следующем онлайн-запуске — service worker подтягивает свежий `program.json`).

## Публикация (GitHub Pages)

Приложение статическое — его можно бесплатно разместить на GitHub Pages:

1. **Settings → Pages → Build and deployment → Source: Deploy from a branch**.
2. Ветка с этим кодом, папка `/ (root)`.
3. Открыть выданный адрес вида `https://<user>.github.io/insonmia/` на телефоне.

> Пути в приложении относительные, поэтому оно корректно работает из подпапки `/insonmia/`.
> Для PWA и уведомлений нужен HTTPS — GitHub Pages его предоставляет автоматически.

## Про уведомления

- Приложение планирует напоминание за выбранное число минут до начала каждого избранного события.
- На браузерах с поддержкой **Notification Triggers** (современный Chrome/Edge на Android) напоминание
  срабатывает по расписанию ОС — **даже если приложение закрыто**.
- На остальных (в т.ч. iOS Safari) напоминание приходит, пока приложение открыто или свёрнуто в фоне.
  Для надёжности перед событием держите приложение установленным на главный экран.

## Технические детали конвертации

События в расписании переходят за полночь (например, показ в блоке «9 июля», начинающийся в 00:30,
реально идёт ранним утром 10 июля). Конвертер и браузерный импортёр вычисляют реальные даты-время:
время с часом меньше 9 утра относится к следующему календарному дню. Это позволяет корректно
сортировать события и показывать «что идёт сейчас».

Идентификаторы событий — детерминированный хеш (FNV-1a) от `тип|дата|время|площадка|название`,
одинаковый в Python и в браузере, поэтому отметки «избранное» сохраняются при обновлении данных.
