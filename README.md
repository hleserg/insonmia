# 🌙 Бессонница 2026 — офлайн-программа

Мобильное веб-приложение (PWA) с программой фестиваля анимации **Бессонница 2026**.
Работает **полностью офлайн**, ставится на главный экран телефона как обычное приложение,
показывает описания событий, что идёт прямо сейчас, избранное и **напоминания за 15 минут**
до начала выбранных событий.

## Что умеет

- 📅 **Программа по дням** (9–13 июля) с фильтрами: дневная программа / ночная анимация.
- 🔴 **«Сейчас»** — что идёт в эту минуту и что будет дальше, с обратным отсчётом до старта фестиваля.
- 📖 **Полные описания** событий и списки фильмов в программах показов.
- ⭐ **Избранное** — отметьте событие звёздочкой; список избранного отсортирован по дате и времени.
- ⏰ **Напоминания** приходят за 15 минут (настраивается: 5/10/15/30/60) до начала избранных событий.
- 🔎 **Поиск** по названию, площадке и фильмам.
- 🔄 **Обновление программы** без переустановки: из новой Excel-таблицы (файл или ссылка)
  или проверкой обновления на сервере.
- 🗺 **Настоящая офлайн-карта** (Leaflet + собственная подложка из данных OpenStreetMap:
  река, лес, луга, тропы — тайлы не используются, их массовую загрузку OSM блокирует):
  зоны площадок, дороги, 150+ точек по категориям (еда, туалеты, души, сцены, арт, медпункт,
  КПП…), фильтры-чипсы, карточка точки с событиями «здесь сегодня», кнопка «я где?» — GPS
  работает без интернета. В карточке события — «📍 на карте» с центровкой и подсветкой зоны.
- 📡 **«Рядом»**: что идёт/скоро в радиусе 150/300/600 м от вас, с метрами и сторонами света.
- 🌃 **Фестивальные сутки**: день длится с 06:00 до 05:59 — ночные показы не разрывает полночью,
  в 02:00 ночи активна ещё «вчерашняя» вкладка; у ночных событий маркер «🌙 ночь на пн»
  и разделитель «после полуночи» в списке.
- 🕐 **Московское время везде**: расчёты «сейчас/скоро» верны, даже если телефон в другой таймзоне.
- 📡 **Гайд «Связь на поляне без интернета»** (`mesh.html`, работает офлайн):
  как поставить Bluetooth-mesh-мессенджер Bitchat до выезда и переписываться в поле,
  плюс обмен метками карты прямо из текста сообщений.
- 📌 **Свои метки на карте** (лагерь, машина, душ): лонгтап по карте или кнопка ➕,
  эмодзи, заметка, лимит 50. Обмен ссылкой без сервера (`#pin=…` — открывший видит
  «Добавить?»), «Добавить из текста» (координаты с русской запятой, geo:-ссылки),
  экспорт всех одной строкой. В «рядом» свои метки — первым блоком.
- 🧪 **Автотесты** вместо ручной симуляции: `npm test` — 35 юнит-тестов логики
  «сейчас»/«рядом»/меток (фестивальные сутки, границы 05:59/06:00, сквозь полночь,
  хаверсин, радиусы, стороны света, watchPosition-lifecycle, парсинг координат,
  диплинки), прогон в 4 таймзонах. Плюс e2e-сьюты в `tests/e2e/` (README внутри).
  Симуляция `?now=`/`?mockgeo=` в проде выключена (флаг `DEV` в app.js — для разработки).
- 📴 **Офлайн-first**: после первого открытия всё живёт в кэше и работает без интернета
  (network-first с таймаутом ~3.5с → тихий фолбэк в кэш; офлайн — штатный режим, не ошибка).
- 📲 **Установка на главный экран** (Android/iOS), запуск в полноэкранном режиме.

## Как пользоваться (для зрителя)

1. Откройте страницу приложения в браузере телефона (см. «Публикация» ниже).
2. Нажмите **⚙️ → «Установить приложение»** (Android) или **«Поделиться» → «На экран Домой»** (iPhone).
3. Один раз откройте при наличии интернета — программа скачается в память телефона.
4. Дальше приложение работает офлайн. Отмечайте события ⭐, разрешите уведомления в настройках —
   и получите напоминание за 15 минут до начала.

## Структура проекта

```
index.html            — разметка приложения
styles.css            — стили (тёмная «ночная» тема)
app.js                — вся логика: рендер, избранное, уведомления, импорт Excel
sw.js                 — service worker (офлайн-кэш + клики по уведомлениям)
manifest.webmanifest  — манифест PWA (установка на главный экран)
data/program.json     — программа фестиваля (генерируется из экспорта сайта)
data/geo.json         — карта: точки/зоны/дороги (из Google My Maps KML)
data/place-aliases.json — ручные алиасы «площадка программы -> точка карты»
data/basemap.json     — подложка карты: вода/лес/тропы из Overpass (fetch-tiles.yml)
map.js                — Leaflet-карта и раздел «Рядом»
vendor/xlsx.full.min.js — SheetJS, чтение Excel прямо в браузере (для офлайн-импорта)
icons/                — иконки приложения
source_xlsx/          — исходные Excel-файлы фестиваля (резервный источник)
scripts/scrape_site.py  — скачивание и конвертация экспорта insomniafest.ru
scripts/convert_xlsx.py — конвертер Excel → program.json (fallback)
tests/fixtures/       — снапшот экспорта сайта для офлайн-разработки
tests/js/             — юнит-тесты core.js (node --test, запуск: npm test)
core.js               — чистая логика времени/гео (работает в браузере и в node)
.github/workflows/update-program.yml — автообновление программы по расписанию
```

## Обновление программы, когда фестиваль поменял расписание

Все пути сохраняют отметки «избранное» (id событий — детерминированный хеш).

**Способ 0. Автоматически (основной).** GitHub Action
`.github/workflows/update-program.yml` раз в 6 часов скачивает официальный экспорт
программы `https://insomniafest.ru/export/program/2026`, конвертирует его скриптом
`scripts/scrape_site.py build` и коммитит `data/program.json`, если данные изменились.
Зрителю остаётся нажать ⚙️ → «Проверить обновление на сервере» (или просто открыть
приложение онлайн — service worker подтянет свежий файл сам).

**Способ 1. Прямо в приложении.**
⚙️ → «Загрузить из Excel-файла…» и выберите скачанный с сайта `.xlsx`
(можно выбрать сразу оба файла — программу и анимацию). Или вставьте прямую ссылку на `.xlsx`.
Парсинг Excel происходит в браузере и работает офлайн.

**Способ 2. Пересобрать встроенные данные вручную.**

```bash
# из экспорта сайта (как это делает Action):
python3 scripts/scrape_site.py build
# либо офлайн из фикстуры:
python3 scripts/scrape_site.py build --from tests/fixtures/export_program_2026.json
# либо из Excel-файлов в source_xlsx/:
pip install -r requirements.txt && python3 scripts/convert_xlsx.py
```

## Публикация (GitHub Pages)

Приложение статическое — его можно бесплатно разместить на GitHub Pages:

1. **Settings → Pages → Build and deployment → Source: Deploy from a branch**.
2. Ветка с этим кодом, папка `/ (root)`.
3. Открыть выданный адрес вида `https://<user>.github.io/insonmia/` на телефоне.

> Пути в приложении относительные, поэтому оно корректно работает из подпапки `/insonmia/`.
> Для PWA и уведомлений нужен HTTPS — GitHub Pages его предоставляет автоматически.

## Про уведомления

- Приложение планирует напоминание за выбранное число минут до начала каждого избранного события.
- На браузерах с поддержкой **Notification Triggers** (современный Chrome/Edge на Android) напоминание
  срабатывает по расписанию ОС — **даже если приложение закрыто**.
- На остальных (в т.ч. iOS Safari) напоминание приходит, пока приложение открыто или свёрнуто в фоне.
  Для надёжности перед событием держите приложение установленным на главный экран.

## Версии и обновления

- В `data/program.json` есть `meta.version` (момент выгрузки с сайта) — она видна в футере
  приложения («Данные с insomniafest.ru от 2 июля, 21:40»). Файл перезаписывается только если
  контент реально изменился — cron не создаёт пустых коммитов.
- При появлении новой версии данных приложение показывает тихий тост «Расписание обновлено».
- При выходе новой версии самого приложения — баннер «Доступна новая версия — обновить»
  (обновление только по клику, без внезапных перезагрузок).
- Внизу приложения — дисклеймер: это неофициальное фанатское приложение; истина в последней
  инстанции — инфоцентр и бумажная карта фестиваля.

## Технические детали конвертации

События в расписании переходят за полночь (например, показ в блоке «9 июля», начинающийся в 00:30,
реально идёт ранним утром 10 июля). Конвертер и браузерный импортёр вычисляют реальные даты-время:
фестивальные сутки длятся с 06:00 до 05:59 следующего дня (`DAY_CUTOFF = 6`), поэтому события
после полуночи относятся к «вчерашнему» фестивальному дню. Это позволяет корректно сортировать
события и показывать «что идёт сейчас».

Идентификаторы событий — детерминированный хеш (FNV-1a) от `тип|дата|время|площадка|название`,
одинаковый в Python и в браузере, поэтому отметки «избранное» сохраняются при обновлении данных.
