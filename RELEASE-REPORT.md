## 4. Отчёт предрелизного аудита (6 фаз, PR #8) и последующих ревью

**Процесс:** после каждой задачи — адверсарное ревью (независимые ревьюеры → находки → каждую пытается опровергнуть отдельный агент чтением кода и запуском репро). Раунды: аудит (51 находка), гео-фикс (11), метки (22), гайд (10), плюс 5 раундов в ранних задачах (36). **Итого 106+ находок — все закрыты фиксом или обоснованным «оставить».**

**Главное, что было починено по раундам:**
- iOS: ReferenceError на голом `Notification` ломал «Избранное»/импорт; отмена OS-напоминаний не работала; поллер терял напоминание при упавшем показе; xlsx-импорт сдвигал время на таймзону устройства.
- Карта рисовалась поверх модальных окон (стекинг Leaflet); кнопка 🎯 воровала тапы у подсказки; протухшая GPS-позиция выдавалась за актуальную.
- Парсер координат: «5,5 км до лагеря: 54,68712 35,07934» рождал метку в океане; формат «координаты, под ними имя» схлопывал метки; пустые координаты в ссылке давали «нулевой остров».
- Гайд: ссылка «назад» уезжала под iOS-статус-бар (в standalone нет кнопки назад — тупик); гонка наложения модальных окон; офлайн `/mesh` без `.html` молча отдавал приложение.
- Пайплайн: фикстура KML писалась до валидации; cron без таймаута; гейты перекрытия id.

**Упрощено:** время/фест-сутки только в core.js; общие хелперы вместо дублей; выпилены мёртвые поля/CSS/ветки; `pinKey`/лимиты — в одном месте.

**Осознанно оставлено:** атрибуция Leaflet 10px (лицензия OSM); «скоро» = ≤30 мин везде; авто-дороги выключены по умолчанию; xlsx 861 КБ в прекэше — цена офлайн-импорта Excel; вкладка «Пн 13» (решение владельца); timeout GPS 15 с — watch продолжает ловить в фоне и сам оживляет список; даты вида «03.07 04.07» парсер меток может принять за координаты — их поймает мягкое предупреждение «далеко от поляны».

## 5. Известные особенности (для поста, человеческим языком)

- **Первый запуск — с интернетом.** Один раз открыть онлайн — дальше всё живёт без сети, обновления подтянутся сами при появлении связи.
- **Ставьте из Chrome.** Надёжный путь: меню Chrome (⋮) → «Установить приложение». Яндекс Браузер капризен с GPS и уведомлениями — в приложении есть подсказка «Не работает геолокация?».
- **Избранное и напоминания живут в установленном приложении**, не во вкладке браузера — вкладку телефон может выгрузить.
- **Свои метки на карте**: лонгтап — поставить (лагерь/машина/душ), из карточки — поделиться ссылкой; открывший добавит метку себе даже офлайн, если приложение уже стоит. Координаты из сообщений вставляются через «Добавить из текста» — русская запятая понимается.
- **GPS в поле работает без интернета**, но первый захват спутников — до пары минут под открытым небом.
- **Напоминания на iPhone** приходят, пока приложение открыто/свёрнуто — iOS не даёт веб-приложениям будить телефон.
- **Карта своя, не Google**: рисованная подложка + 150+ точек с фестивальной карты; «серых квадратов» офлайн не бывает.
- **Время всегда московское**, даже если телефон в другой таймзоне.
- **Ночные показы не рвутся полуночью**: день длится до 06:00, показ в 02:00 — в программе «вчерашнего» дня с маркером «🌙».
- **Связь на поляне**: гайд по Bitchat (Bluetooth-mesh) внутри приложения, работает офлайн; сам Bitchat поставить ДО выезда.

## 6. Памятка владельцу

- **Автообновление данных:** каждые 6 ч CI тянет экспорт сайта и коммитит `data/program.json` → Pages → приложение показывает тост. Руками: Actions → «Обновление программы» → Run workflow.
- **Хотфикс кода:** ветка → PR → merge в main; **обязательно поднять `CACHE` в `sw.js`** — иначе клиенты останутся на старой версии; у пользователей появится баннер «обновить».
- **Экстренно данные без CI:** `python3 scripts/scrape_site.py build` → коммит `data/program.json` (гейт не пустит мусор; форс — `INSOMNIA_FORCE=1`).
- **Структура в 5 строк:** `app.js` — логика UI; `core.js` — чистое время/гео/метки (покрыто тестами); `map.js` — карта, «рядом», метки; `sw.js` — офлайн-кэш (версия `CACHE`!); `data/*` — программа/гео/подложка из `scripts/*`.
- **Перед фестивалем:** прогнать `node tests/e2e/offline.js` и проверить прод с телефона в авиарежиме; из очереди осталось: иконка (#28+#32) и кнопка установки с плашкой (#30).
